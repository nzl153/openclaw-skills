# Novel Writer Skill

## 🔥 写作铁律

1. 🥇 **先读锁定方向** — 剧情进度文件的「当前章计划」有锁定续写方向就直接用，不每天一个新想法
2. 🥇 **写大纲前先查历史记录** — 确认用户之前讨论过的决定，不凭印象瞎编
3. 🥇 **不确定先问用户** — 涉及信息边界（谁知道了什么）、结尾场景衔接、角色位置，先确认再写
4. 🥇 **原稿必须存** — 每写一章就把定稿原文存到 `原稿/第X章.md`
5. 🥇 **章节摘要至少100字** — 包含关键事件、冲突点、角色动向、结尾场景
6. 🥇 **不编名字/设定** — 角色名、门派名、地名全部先查设定文件确认是否已有，没有的先问用户
7. ⚠️ **写完必须三查**：
   ① 查名字/设定——有没有自己编的、不在设定文件里的？
   ② 查实力匹配——角色的出手、对位是否符合设定品级？
   ③ 查信息边界——每个角色知道什么、不知道什么？
8. ⚠️ **带入角色逻辑** — 写每段前想三秒：这个角色在这个场景下会说什么、做什么？符合他的性格、实力、掌握的信息吗？
9. ⚠️ **写完原文后重读一遍** — 不急着交，通读检查有没有逻辑矛盾、前后不一致的地方
10. ⚠️ **设定统一管理** — 所有门派、人物、品级、地名一次性写在设定文件里，不东一个西一个

## Core Workflow

### Session Start: Load Story State

At the start of each writing session, **按层级顺序读取存档**：

```
references/《书名》/
```

读取顺序（层级式，不跳步）：
1. **第一层·必读少量**: 已纠正错误.md → 设定速查.md → 剧情进度.md（重点看当前章计划是否有锁定方向）
2. **第二层·必读略多**: 角色档案.md → 章节摘要.md（近3章·百字版）→ 原稿/上一章.md（完整原文）
3. **第三层·按需**: 时间线.md / 更早的章节摘要 / 更早的原稿

> ⚠️ **铁律**: 写新章方向前必须先查历史对话记录确认用户讨论过的决定，不凭印象瞎编

### Chapter Writing

0. **先读锁定方向**: 读剧情进度.md的「当前章计划」→ 如果已经有锁定续写方向，直接按那个走
1. **查历史确认讨论**: 写任何大纲/续写方向之前，先查对应日期的历史记录，确认用户之前讨论过什么决定
2. **Load context**: 按层级顺序读存档 → 确认当前写到哪了
3. **确认用户**: 不确定的事（结尾场景、角色位置、信息边界）先问用户确认
4. **Read Previous Chapter Original**: 读原稿/上一章的完整原文，确保接续准确
5. **Review tracking**: 扫一眼角色表和伏笔清单
6. **查设定文件**: 写任何新名字（角色、门派、地名）前，先查设定文件确认是否已有
7. **带入角色逻辑 — 三问**：
   ① 这个角色现在的实力等级是什么？出手/对位会不会矛盾？
   ② 他知道什么信息？不知道什么信息？
   ③ 以他的性格，在这个场景下会怎么做？
8. **Write draft chapter** (target: 2000-4000 characters for 番茄)
9. **写完自检 — 三查**：
   ① 查名字——有没有自己编的、不在设定文件里的？
   ② 查逻辑——角色行为、实力对位、信息边界有没有漏洞？
   ③ 查前后矛盾——整体读一遍，有没有和前后文矛盾的地方？
10. **User reviews** → iterate
11. **Update存档**: 更新剧情进度.md + 章节摘要.md + 角色档案.md + 原稿/第X章.md

### Chapter Structure (番茄)

Each chapter should follow:

```
【开篇钩子】→ 第一段制造悬念/冲突/爽点，抓住滑屏读者
【中段推进】→ 推进剧情，穿插爽点/笑点/金手指展示
【结尾悬念】→ 留钩子（突破前、大战前、秘密揭晓前断章）
```

- 番茄读者是"滑屏"阅读，前100字决定留存
- 段落短（2-4句一段为宜）
- 对话多、描写精炼
- 每章至少一个爽点/笑点/悬念

## 📁 存档系统（长篇小说专用）

几十万字的长篇小说，**一个存档文件不够**。
每本书在 `references/` 下创建一个以书名命名的目录，拆成多个文件：

```
references/
  《书名》/
    ├── 0-设定速查.md          # 世界观、力量体系、核心规则（基本不变）
    ├── 1-角色档案.md          # 角色详细表（含实力变化记录）
    ├── 2-剧情进度.md          # 当前章节计划（含锁定续写方向）、未解伏笔、待办
    ├── 3-章节摘要.md          # 每章100字以上剧情摘要+结尾场景
    ├── 4-时间线.md            # 小说内日期/天数的变化
    ├── 5-已纠正错误.md        # ⚠️ 写作前必读！用户纠正过的坑
    └── 原稿/                  # ★ 每章完整原文存档
        ├── 第1章.md
        ├── 第2章.md
        └── ...
```

### 写作前必读顺序（层级式）

每次新会话开始写本章前，**按这个顺序读**，不跳步：

```
第一层：【必读·少量】
  ① 5-已纠正错误.md          ← 30秒扫完，确认没再犯过的坑
  ② 0-设定速查.md            ← 1分钟扫完世界观和力量体系
  ③ 2-剧情进度.md → 当前章节计划  ← ★ 重点！看是否有锁定续写方向

第二层：【必读·略多】
  ④ 1-角色档案.md → 角色表  ← 扫角色表和短期变动群
  ⑤ 3-章节摘要.md → 前3章   ← 读最近3章百字摘要
  ⑥ 原稿/上一章.md           ← ★ 读完整原文，确保接续准确

第三层：【按需查阅】
  ⑦ 4-时间线.md              ← 需要确认小说内天数时再读
  ⑧ 更早的章节               ← 需要查前文伏笔时再翻
  ⑨ 历史对话记录             ← 🔥 写新章方向前必须查！
```

## 写作原则

### 各流派写法要点

**穿越/重生修仙玄幻:**
- 第一章靠五感描写立住穿越感，金手指不要急着出
- 修炼体系通过打斗和突破自然展示，不写说明文档
- 战力系统保持一致性，避免越到后面越崩
- 金手指有成长线，不是一开始就无敌

**武侠:**
- 体系更稳定，不容易战力崩坏
- 内力+招式+轻功+兵器，读者天然理解
- 武林门派、正邪对立，天然剧情驱动力
- 秘籍/兵器可以成为章节钩子和爽点载体

**都市/现代:**
- 主角的生活细节要真实（工作、租房、收入），让读者共情
- 金手指要和现实生活产生化学反应
- 职业装逼/财富逆袭/身份暴露是核心爽点

**系统流/签到流:**
- 第一章就要让读者明白系统规则
- 首奖要香，建立期待感
- 系统有成长性，后期会升级解锁新功能
- 系统面板不要每章都贴，偶尔出现即可

**游戏/电竞:**
- 让读者快速理解是什么游戏类型
- 操作描写要有真实感
- 游戏内成就影响现实生活

**科幻/末世:**
- 第一章快速建立世界观
- 生存压力拉满
- 人性刻画是核心看点

### 番茄平台注意事项
- 前3章决定留存率，开局要有吸引力
- 每章2000-4000字为宜
- 断章在爽点/悬念处，保持追读
- 更新频率建议日更
