# Verify Numbers - 数字验证技能

## 核心原则

**铁律：所有数字先算再答，不凭印象、不嘴瓢。**

你有一个已知的缺陷：容易在数字/时间/日期相关问题上脱口而出错误数值，已经被用户多次纠正。此技能的目的是**用机制代替自觉**，强制你走验证流程。

## 工作流程

### 标准流程

当触发此 skill 时，必须依次执行以下步骤：

**Step 1: 获取当前准确时间**
```
session_status → 获取当前日期和时间（Asia/Shanghai 时区）
```

**Step 2: 查记忆中的相关数据**
```
memory_search → 搜索用户问题涉及的相关信息
（比如问"多久"就搜睡觉时间、问"几天"就搜日记起始日等）
```

**Step 3: 用工具计算**
根据问题类型选择验证方式：

| 问题类型 | 验证方式 |
|---------|---------|
| 时间差/天数 | `exec date` 或 shell 脚本计算 |
| 具体数字 | 搜索或查文件确认 |
| 课表/日程 | 查 MEMORY.md 或 USER.md |
| 价格/预算 | 查记忆中的预算记录 |
| 比例/百分比 | 用 exec 计算器确认 |

**Step 4: 对比再回答**
```
如果计算得出的数字  ≠  姐姐脑子里冒出来的数字
    → 以计算结果为准
    → 永远不输出脑子里冒出来的未经计算的数字
```

**Step 5: 回答时附加验证说明（可选）**
如果用户有追问需求，可以附带来源引用。

### 特殊情况

**用户纠正数字错误时：**
1. 立刻停止回答
2. 承认错误（不要找借口）
3. 重新走标准流程 Step 1-4
4. 给出正确数值

**用户要求重新计算时：**
1. 清空之前的数字结果
2. 重新走标准流程
3. 只输出新结果

## 触发关键词

当用户消息中包含以下关键词或其近似表达时，必须激活此 skill：

- 时间类：几天、几小时、几分钟、几点、几号、什么时候、多久、还剩、距离
- 数字类：多少、几个、几块、多少钱、百分之、比例、倍数、对比
- 统计类：总共、累计、平均、总共花了、花了多少
- 纠正类：错了、不对、你算错了、你再算算
- 日期类：从……到、至今、距离……还有、已经过了

## 已知失败案例（警示）

- ❌ 问"5月12号到现在多少天"，姐姐答"13天"（实际6天）
- ❌ 类似的时间计算错误已发生多次

## 本质

这不是一个复杂的技能。它不需要脚本、不需要外部工具、不需要额外资源。

**它只是一个强制流程——让姐姐在回答数字问题之前，先动手验证，而不是动嘴输出。**

用机制代替自觉。
